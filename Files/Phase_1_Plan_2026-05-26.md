## Scope

Replace the `UserDefaults`-backed mock `AuthService` with a real Firebase Auth-backed implementation, keeping the public UI-facing API stable so `RootView`, `AuthView`, and `SettingsView` need only small touch-ups.

**Not in this phase:** Firestore push/pull (Phase 4), StoreKit, Cloud Functions, account deletion server-side cascade.

## Decisions (locked 2026-05-26 PM)

1. **Auth model: mandatory, Apple-only.** No anonymous, no Google, no email path in Phase 1. `AuthView` is already Apple-only in its UI — matches.
2. **Email: not in Phase 1.** No email link, no email/password. Defer until a real need surfaces post-MVP.
3. **Account deletion: local + Auth only.** `Auth.currentUser.delete()` + wipe local SwiftData via the Phase 0 repositories (which enqueue tombstones for Phase 4 sync). Cloud Function cascade lands in Phase 4 with Firestore. MVP passes App Store 5.1.1(v) because there's no remote data attached yet.

## Code changes Claude will make in Phase 1

### 1. `Core/Services/AuthService.swift` — replace mock body, keep public API

Two changes:
- Introduce a private `AuthServicing` protocol-ish boundary (or keep concrete `AuthService` as a facade — Codex audit suggests the facade for speed; lean on that).
- Replace `UserDefaults` restore with a Firebase auth state listener:

  ```swift
  init() {
      handle = Auth.auth().addStateDidChangeListener { [weak self] _, user in
          self?.account = user.map { AccountSummary(from: $0) }
      }
  }
  deinit { Auth.auth().removeStateDidChangeListener(handle) }
  ```

- Implement `signInWithApple()` via `ASAuthorizationAppleIDProvider` + `OAuthCredential` exchange against Firebase.
- **Delete** `signInWithEmail(_:)`, `signInWithGoogle()`, `signInAsGuest()`. Mandatory Apple-only — these are dead code under the new model.
- Keep `signOut()` synchronous (`try Auth.auth().signOut()`).
- Add `deleteAccount()` that calls `Auth.auth().currentUser?.delete()` then iterates each repo to delete the user's local records (so Phase 4 will get tombstones enqueued). Wires into [SettingsView.swift:80](DevCal/DevCal/Features/Settings/SettingsView.swift#L80).
- Surface errors: convert `signInWithApple()` and `deleteAccount()` to `async throws`; views handle via the same `systemAlert` pattern Phase 0 added.

### 2. `DevCalApp.swift` — bootstrap Firebase

- Add `import FirebaseCore` + `import FirebaseAuth` at top.
- In `init()`, before constructing `ModelContainer`: `FirebaseApp.configure()`.
- No email-link `.onOpenURL` handler — email not in Phase 1.

### 3. `Features/Auth/AuthView.swift`

- File is already Apple-only (single button) — no UI changes needed.
- Wrap the Apple button's action in `Task { try await auth.signInWithApple() }` with try/catch → error alert. Currently calls a non-throwing `async`; flipping to throws is a tiny ripple.
- Keep the `isWorking` guard around the button so double-tap is impossible.

### 4. `Features/Settings/SettingsView.swift:80` — wire account deletion

Replace the `TODO(firebase)` comment with `try await auth.deleteAccount()` inside a confirmation `systemAlert`. Add a copy line noting local data is also wiped (use `ModelContext.deleteAllData()` or iterate and delete via repositories — preferred to enqueue tombstones for Phase 4 sync).

### 5. `Features/Settings/CloudSyncSettingsView.swift:145` — leave alone

That's the "Sync now" button → Phase 4 hookup. Phase 1 still no-ops via `NoopSyncService`.

# DevCal 統計資訊規劃

Last updated: 2026-05-21

## 核心原則

DevCal / SideProfit 的統計資訊只回答四個問題：

1. 這個專案有沒有回本？
2. 這個月表現是變好還是變差？
3. 錢主要流到哪裡？
4. 我應該把時間繼續投入哪個專案？

設計規則：

- 專案首頁看健康度。
- 專案分析看原因。
- 洞察頁看取捨。
- 現金淨利永遠不扣除時間成本。
- 時間成本獨立呈現，避免把 indie project 都算成永久失敗。
- 避免圓餅圖。分類占比用排行列與比例條呈現。
- 避免泛用財務語言，例如「交易」、「資產」、「負債」、「現金流量表」。

## 頁面分工

### 1. 專案列表

目的：快速比較所有專案，決定要點進哪一個。

每個專案列顯示：

| 資訊 | 用途 |
| --- | --- |
| 專案名稱 | 識別 |
| 專案狀態 | Planned / Building / Live / Paused |
| 本月淨利 | 最近表現 |
| 累積淨利 | 長期結果 |
| 回本 / 目標進度 | 是否正在變成真的事業 |

不放：

- 支出分類
- 收入分類
- 詳細月表
- 跨專案排行

### 2. 專案首頁

目的：回答「這個專案現在健康嗎？」

建議區塊：

| 區塊 | 內容 |
| --- | --- |
| Hero | 回本進度 / 目標進度 |
| 核心數字 | 累積收入、累積支出、累積淨利 |
| 月度趨勢 | 最近月份的淨利摘要 |
| 收支紀錄 | 最近 5 筆收入 / 支出 |
| 時間成本 | 總時數、實質時薪，Pro gate |

命名：

- 區塊名稱：月度趨勢
- CTA：查看分析
- 目的地：專案分析

### 3. 專案分析

目的：回答「為什麼這個專案變好或變差？」

目前規劃區塊：

| 區塊 | 內容 |
| --- | --- |
| 月度表現 | 每月收入 vs 支出 chart、月度明細表 |
| 累積進度 | 累積收入、累積支出、累積淨利、距離回本 / 目標 |
| 成本排行 | 支出分類金額、占比、比例條 |

月度明細表欄位：

| 欄位 | 說明 |
| --- | --- |
| 月份 | 該月 |
| 收入 | 該月收入 |
| 支出 | 該月支出 |
| 淨利 | 收入 - 支出 |
| 狀態 | 獲利 / 虧損 / 打平 |

未來可加：

- 與上月相比
- 累積淨利欄位
- 近 3 個月平均收入
- 預估回本月份

### 4. 洞察

目的：回答「所有專案加總後，整體表現如何？哪個專案最值得繼續投入？」

洞察頁是 portfolio view，不是單一專案分析頁。

Toolbar 篩選：

| 選項 | 說明 |
| --- | --- |
| 本月 | 只統計本月收入、支出、時間與淨利排行 |
| 本年 | 只統計本年收入、支出、時間與淨利排行 |
| 全部 | 預設值，從專案建立到現在的累積資料 |

未來可加：自定義時間區間。

頂部指標：

| 指標 | 說明 |
| --- | --- |
| 區間總淨利 | 依 toolbar 篩選後的所有專案現金淨利 |
| 區間總收入 | 依 toolbar 篩選後的所有專案收入 |
| 已回本 | 已達回本的專案數 / 總專案數，維持生命週期統計 |
| 平均時薪 | 區間現金淨利 / 區間總時數 |

主列表改為「淨利排行榜」，依目前 toolbar 區間的淨利高到低排序，摘要只顯示前五名。若超過五個專案，右側顯示「全部」，進入完整排行榜頁。

每列顯示：

| 資訊 | 說明 |
| --- | --- |
| 排名 | #1 到 #5 |
| 專案名稱 | 點擊進入專案首頁 |
| 區間淨利 | 主要排名數字，依目前 toolbar 篩選重算 |

排序：

1. 目前 toolbar 區間淨利高到低
2. 區間淨利相同時，累積淨利高到低
3. 仍相同時，依專案名稱排序

合併版分析區塊：

| 區塊 | 內容 |
| --- | --- |
| 月度表現 | 所有專案合併後的每月收入 vs 支出 chart |
| 月度明細 | 所有專案合併後的月份、收入、支出、淨利、狀態 |
| 累積進度 | 所有專案合併後的收入、支出、淨利、距離回本 |
| 成本排行 | 所有專案合併後的支出分類金額、占比、比例條 |

這些區塊都受 toolbar 時間範圍影響；「全部」代表從專案新增至今的累積總覽。

洞察頁鑽取規則：

| 入口 | 行為 |
| --- | --- |
| 淨利排行榜「全部」 | 進入完整排行榜頁，顯示所有專案排名、專案名稱、區間淨利；整列可進入專案首頁 |
| 累積收入 / 累積支出 / 累積淨利 / 距離回本卡片 | 開啟全高 sheet，以排行榜列型顯示排名、專案名稱、該卡片對應的單一指標；整列可進入專案首頁 |
| 成本排行列 | 開啟全高 sheet，以排行榜列型顯示使用該成本分類的專案與金額，依金額由高到低排序；整列可進入專案首頁 |
| 成本排行「全部」 | 進入完整成本分類排行頁；分類列仍可再開啟對應專案明細 sheet |

洞察首頁只放摘要與前五名；完整列表、專案拆解與成本拆解放到下一層，避免手機版出現橫向捲動表格。全高 sheet 不允許下滑關閉，左上角提供非品牌色叉叉按鈕。

### 5. 時間成本

目的：回答「如果把我的時間算進來，這個專案值得嗎？」

時間成本頁只處理時間，不重新定義現金淨利。

建議指標：

| 指標 | 說明 |
| --- | --- |
| Total hours | 總投入時數 |
| Hidden cost | 時數 × 使用者設定時薪 |
| Net incl. labor | 現金淨利 - Hidden cost |
| Effective rate | 現金淨利 / 總時數 |

## 不採用的呈現

- 圓餅圖 / donut chart
- 每頁重複同一組收入、支出、淨利
- 沒有足夠資料時硬算 MoM
- 把時間成本混進現金淨利
- 會讓產品變成一般記帳 app 的財務報表
